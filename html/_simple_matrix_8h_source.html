1 /*
2  * SimpleMatrix.h
3  *
4  * This involves a gambit very similar to the VecBase scenario:
5  * We have a class which is essentially unchanging in its body, but
6  * needs to have permuted constructors based on a template parameter.
7  * Therefore we put the entire body in a base class, and then make a thin
8  * derived class with partial specialization, and design the constructor
9  * interface as we please for each specialization.
10  *
11  * This class structure is pretty darn ugly, so we hide it from the user
12  * for the purposes of Doxygen by simply pretending the (hidden) common base class
13  * is the user-facing one.
14  *
15  * Created on: Apr 18, 2013
16  * Author: tbabb
17  */

29 /*
30  what would we need for templated layout?
31 
32  - tell us the row iterators
33  - tell us the column iterators
34  - provide get() --> {const,} iterator
35  - tell us how create storage
36  - tell us how to fill(?)
37 
38  Would need to update generic row and column iterators to
39  iterate over the whole matrix, not just their own column.
40  This would mean that row() or col() would behave the same
41  way regardless of layout.
42 
43  This would allow us to more easily treat vectors as columns
44  directly; e.g. for linear solve, tracing triangle, orthogonalizaing...
45 
46  Might be even better to abstract the memory into a Grid<...>.
47 */
