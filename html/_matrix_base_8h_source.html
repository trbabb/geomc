23 /*
24  * Matrix typedefs:
25  *
26  * elem_t (T)
27  * reference (T& or AssignmentProxy)
28  * iterator (T* or SubsetIterator)
29  * const_iterator (const T* or SubsetIterator<const T>)
30  * row_iterator (T* or SliceIterator or StrideSliceIterator)
31  * const_row_iterator (const T* or SliceIterator<const> or StrideSliceIterator<const>)
32  * col_iterator (T* or SliceIterator or StrideSliceIterator)
33  * const_col_iterator (const T* or SliceIterator<const> or StrideSliceIterator<const>)
34  * region_iterator (SubsetIterator)
35  * const_region_iterator (SubsetIterator<const>)
36  *
37  * constant iterators cannot be written through.
38  * - They accomplish this by returning `const T` objects, which are not lvals.
39  * - Non-constant iterators, by contrast, return references (or proxy references) to T objects.
40  * - Constant iterators shall call the const version of m.get(), which by nature
41  * does not return writable references. This is done by passing `const M` as
42  * the matrix type to their templates.
43  */

48 // here we use the curiously recurring template pattern
49 // to provide "template inheritance", i.e. the ability
50 // for a whole host of distinct template classes to statically
51 // inherit functionality from a "base" template, without virtual
52 // inheritance and all the runtime costs associated with it.
53 
54 // this creates a bizarre (but useful) case where the base
55 // inherits functionality from the derived class.
