Detailed Description

template<typename NodeItem, typename LeafItem>
template<bool Const>
class geom::Tree< NodeItem, LeafItem >::NodeIterator< Const >

An optionally-const iterator over the internal nodes of a tree.

Dereferencing this iterator produces a NodeItem object.

This class can be treated as a container of objects of its own type.

Template Parameters
ConstWhether this iterator refers to a const item or not.

item_iterator erase ( const item_iterator &  item)
inline

Remove an item from this subtree.

If item is not in the subtree, the tree will be unchanged.

Calling this method on a close ancestor of item reduces the cost of finding item's parent in the tree before removal. In general, an O(n * log(n)) search is performed to find the direct parent of item, where n is the size of this subtree.

Invalidates the iterator to this item, as well as any end() item_iterators.

Returns
An iterator pointing to the position of the item just beyond the one that was deleted. This will be parent.items_end() if the deleted item was the last one in its parent node.

Insert a new tree node under this one, to the left of insert_before.

If this node was previously empty, then its new first child will contain all of its LeafItems (this is to ensure those leaf items all have a parent which is a leaf node). Otherwise, the new node will be empty of any LeafItems.

If insert_before is the root node, or if insert_before is not a direct child (or end-child) of this node, then the tree is unaffected and end() is returned. Otherwise, the new node is returned.

Insert an item under this node.

The new item will be inserted before the item at insert_before. insert_before must belong to this node, otherwise the behavior is undefined. It is permissible for insert_before to be this node's items_begin() or items_end().

This must be a leaf node; i.e. one with no child nodes, so that placement of the new objects is not abiguous. If this is not a leaf node, the tree is unchanged.

◆ split()

node_iterator split ( node_iterator &  node,
Func  compare,
P  pivot,
NodeItem &  obj 
)
inline

Split node into two sibling nodes.

A new node will be created to the left of node under the same parent, and the items of node will be split between them according to the result of compare(item, pivot): If the comparison is less than zero, the item will be moved to the left (new) node. Otherwise, it will remain in the right (existing) node.

The split node must not be the root and must have no children. If either is the case, the tree will not be changed and the off-end node will be returned.

All iterators to the items under node are invalidated by this operation, as well as all end() item_iterators.

Parameters
nodeThe node to split; a non-root node with no child nodes.
compareA callable object compare(a,b) which accepts a LeafItem as its left argument and a P as its right argument, and returns a signed number (negative for a < b, positive for a > b, zero for equality).
objThe internal node object to assign to the newly-created (low) node.
Returns
The newly created sibling, or the off-end node if node is the root node or has child nodes.
